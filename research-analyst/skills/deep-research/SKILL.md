# Deep research (method)

Anyone can search; the value is in *how*. This is the method that makes a
web-researched answer trustworthy.

## 1. Decompose the question
Break the ask into 3-5 sub-questions and distinct **angles**. E.g. "Should we
worry about Competitor X?" →
- What have they shipped, and when? (product)
- Who's funding them / how much runway? (financial)
- What are customers saying? (sentiment)
- Who's leaving/joining? (talent signal)
Different angles surface different truths; one query answers none of them.

## 2. Search each angle (multiple queries)
For each sub-question, run several **web-search** queries — vary the phrasing,
add site/date qualifiers, search for the *counter*-claim too. Don't stop at the
first page of the first query.

## 3. Vet every source (before you trust it)
For each source, judge (see `research-standards.md`):
- **Primary vs. secondary** — the original filing/study/announcement, or someone
  summarizing it? Trace to primary.
- **Credibility** — who published it, and do they have a reason to spin?
- **Recency** — is it current, or stale data being re-quoted?
- **Independence** — three sources that all cite one origin are *one* source.

## 4. Triangulate + set confidence
- **High** — multiple *independent*, credible sources agree.
- **Medium** — a single credible source, or limited corroboration.
- **Low** — a lead: one weak/unverified source. Label it, don't rely on it.
When sources **disagree**, surface the conflict and say which you trust + why.

## 5. Look for the counter-evidence
Actively search for what would *disprove* the expected answer. Confirmation bias
is the main way research goes wrong — I try to break my own finding.

## Rules
1. **Decompose first** — angles, not one query.
2. **Triangulate** — one source is a lead, not a fact.
3. **Trace to primary** — don't cite the article that cited the study.
4. **Confidence on every finding** — high/medium/low, honestly.
5. **Hunt the counter-evidence** — prove yourself wrong before you conclude.
